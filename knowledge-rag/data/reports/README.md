# Demo source documents

A small set of source files in every format the ingestion pipeline
supports — drop new files alongside them or replace them with your own
corpus. The companion `.json` files are sidecar metadata that gets
merged into each chunk's `payload`.

| File                           | Format | What it demonstrates                                       |
|--------------------------------|--------|------------------------------------------------------------|
| `employee_handbook.md`         | .md    | Headings → sections; Markdown table is extracted as `ParsedTable`. |
| `release_notes_v2.4.txt`       | .txt   | Plain-text blocks split on blank lines; ALL-CAPS headings detected. |
| `product_announcement.html`    | .html  | `<nav>/<footer>` are stripped; the `<table>` becomes a `ParsedTable`. |
| `quarterly_review_2026Q1.docx` | .docx  | Generated by `tests/fixtures/create_fixture.py`; shows table-row atomicity. |

PDF coverage is exercised by the e2e test suite (which mocks pdfplumber)
and by your own PDFs. Drop any `.pdf` into this directory and it will
be picked up on the next `make ingest`.

## Try it

```bash
# Bring up Qdrant (dev compose)
docker compose up -d qdrant

# Ingest everything in this directory
make ingest

# Launch the chat UI
make ui
```

Sample questions to ask:

- *"How many PTO days can a full-time AMER employee accrue per year?"*
  (Should cite `employee_handbook.md` page 1, section 1.)
- *"Compare the Starter and Team pricing tiers."*
  (Should pull rows from the HTML pricing table.)
- *"What's new in Knowledge RAG 2.4?"*
  (Should pull from `release_notes_v2.4.txt` and the HTML announcement.)

Activate the chemistry example pack to see how a domain pack changes
behaviour:

```bash
DOMAIN_PACK_PATH=domain/examples/chemistry.yaml make ui
```

Then ask a question about a chemical procedure — the prompt set will
switch to the chemistry-tuned persona.
